◆ VerifyOrDie

#define VerifyOrDie (   aCondition)    nlABORT(aCondition)

This checks for the specified condition, which is expected to commonly be true and forces an immediate abort if the condition is false.

Example Usage:

void FreeBuffer(const uint8_t *buf)
{
VerifyOrDie(buf != NULL);
free(buf);
}

[in]aConditionA Boolean expression to be evaluated.

This checks for the specified condition, which is expected to commonly be true and both prints aMessage and forces an immediate abort if the condition is false.

Example Usage:

void FreeBuffer(const uint8_t *buf)
{
VerifyOrDieWithMsg(buf != NULL, MemoryManagement, "Invalid pointer passed to FreeBuffer");
free(buf);
}
Parameters
[in]aConditionA Boolean expression to be evaluated.
[in]aModuleA chip LogModule short-hand mnemonic identifing the logical section of code that is a source the logged message.
[in]aMessageA pointer to a NULL-terminated C string with C Standard Library-style format specifiers containing the log message to be formatted and logged.
[in]...A variadic argument list whose elements should correspond to the format specifiers in aMessage.

This is invoked when a VerifyOrDie or VerifyOrDieWithMsg assertion expression evaluates to false.

Developers may override and customize this by defining chipDie before CodeUtils.h is included by the preprocessor.
